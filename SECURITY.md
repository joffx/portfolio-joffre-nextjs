# Política de Seguridad

## Versiones Soportadas

Actualmente mantenemos y proporcionamos actualizaciones de seguridad para las siguientes versiones:

| Versión | Soportada          |
| ------- | ------------------ |
| 1.3.x   | :white_check_mark: |
| < 1.3   | :x:                |

## Reportar una Vulnerabilidad

Si descubres una vulnerabilidad de seguridad, por favor **NO** crees un issue público. En su lugar, sigue estos pasos:

1. **Contacto Directo**: Envía un email a **joffre@fastery.dev** con el asunto `[SECURITY]` seguido de una breve descripción.

2. **Información Requerida**:
   - Descripción detallada de la vulnerabilidad
   - Pasos para reproducir el problema
   - Impacto potencial de la vulnerabilidad
   - Sugerencias de solución (si las tienes)

3. **Tiempo de Respuesta**: Nos comprometemos a responder dentro de 48 horas y a proporcionar una actualización sobre el progreso dentro de 7 días.

4. **Recompensas**: Actualmente no ofrecemos un programa de recompensas por errores, pero agradecemos enormemente cualquier reporte responsable.

## Proceso de Divulgación

1. Recibiremos y confirmaremos tu reporte
2. Investigaremos la vulnerabilidad
3. Desarrollaremos y probaremos un parche
4. Publicaremos una actualización de seguridad
5. Te notificaremos cuando la vulnerabilidad haya sido resuelta

## Buenas Prácticas de Seguridad

- No compartas credenciales o tokens de acceso
- Mantén tus dependencias actualizadas
- Revisa regularmente los logs de seguridad
- Usa variables de entorno para información sensible
- Implementa autenticación y autorización adecuadas
